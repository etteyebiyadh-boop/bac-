# Engagement and Monetization Blueprint

## 1. Gamification system

### Design rule
Gamification must reinforce disciplined study behavior, not distract from it.

### Core loop
1. student receives a daily mission
2. student completes one task
3. app gives instant feedback and visible progress
4. student earns XP, streak credit, and a recommendation for the next task
5. next session starts with lower friction

### Game mechanics
- `XP`: awarded for lessons, exercises, essays, and mock exams
- `streaks`: daily study consistency tracker
- `levels`: readiness bands such as `Starter`, `13+/20`, `15+/20`, `18+/20`
- `badges`: milestones like first corrected essay, 7-day streak, 5 mock exams
- `weekly challenges`: targeted bundles such as "2 essays + 3 grammar drills"
- `leaderboards`: anonymous ranking by school or cohort
- `streak freeze`: one weekly recovery token for missed days

### Reward balance
- reward completion and consistency more than raw score
- do not create public shame mechanics for low-performing students
- celebrate score improvement, not just high absolute scores

## 2. Daily engagement plan

### Daily mission examples
- complete one grammar pack
- rewrite one weak paragraph from yesterday's essay
- finish one timed bac prompt introduction
- review one smart lesson and solve the recap drill

### Notification strategy

#### Morning
- "Today's 12-minute mission is ready."

#### Afternoon
- "You are one task away from protecting your streak."

#### Evening
- "Your weakest area is connectors. Finish one quick drill before midnight."

### Triggered reactivation
- if the student misses 2 days: send a low-friction comeback task
- if the student hits a new best score: prompt another essay while motivation is high
- if free corrections run out: trigger premium upgrade after result view, not before

## 3. Monetization model

### Free plan
- account creation
- limited exam access
- one daily exercise pack per day
- 3 essay corrections every 7 days
- basic progress dashboard

### Premium plan
- unlimited or high-cap correction bundle
- full bac exam archive
- full smart lesson library
- timed mock exams
- advanced analytics and weak-skill insights
- exam countdown plans and personalized recommendations

### Pricing strategy
- weekly pass for late-season urgency
- monthly subscription as default
- exam-season bundle as highest-LTV plan
- school bulk pricing after consumer traction is proven

### Best paywall moments
- after the student sees a useful corrected essay
- when the student tries to unlock another correction
- when the student wants a premium mock exam

### Weak paywall moments
- before first correction
- immediately at signup
- after low-value actions like browsing lessons only

## 4. Conversion mechanics

### High-intent upgrade triggers
- "You improved by 1.2 points since your first essay. Unlock the next 10 corrections."
- "Your weakest skill is structure. Premium unlocks a personalized weekly plan."
- "You finished your free correction quota. Keep momentum today."

### Discount mechanics
- first premium purchase discount during bac season
- referral credit for inviting classmates
- school code discounts for class partnerships

## 5. Retention instrumentation

### Events to track
- signup completed
- first essay submitted
- correction result viewed
- daily mission completed
- streak protected
- premium paywall viewed
- premium started
- lesson recommended
- lesson completed

### Core dashboards
- activation funnel
- correction usage and repeat rate
- daily mission completion
- conversion by trigger surface
- retention by language and section
